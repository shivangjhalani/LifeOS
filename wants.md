# LifeOS Wants

Core feeling to generate: dump thoughts quickly, trust they are safely structured, and come back to them in a way that feels calm, useful, and personal.

## Stuff

1. Make audio journaling effortless for both quick captures and long reflections.
2. Generate a strong post-journal summary that is useful both immediately and later for recall.
3. Build search around the current validated setup:
   - Layer 1: summary retrieval to answer "which journal?"
   - Layer 2: transcript retrieval to answer "where exactly?"
4. Add better organization views:
   - Timeline view: journal as the atomic unit
   - Topic / thread view: multiple ideas extracted from a single journal
5. Start tracking open loops across journals:
   - unresolved questions
   - decisions in progress
   - topics revisited later

## Ideas To Think about

1. Memory layers, not one giant blob:
   - episodic memory = journal entries and transcript passages
   - semantic memory = stable facts, people, projects, preferences
   - open loops = unresolved questions, tensions, ongoing decisions
2. Citation-first memory:
   - every important memory should link back to the journal and exact passage that supports it
3. Promote durable memories:
   - not everything should stay as raw transcript
   - some things should become reusable long-term memory
4. Temporal linking:
   - detect when a later journal answers or updates an earlier one
5. Optional schemas by journal type:
   - self-reflection, quick capture, gratitude, idea dump, emotional processing
   - shared core fields, plus optional fields depending on the entry

## What The Current Experiments Already Validate

1. Summaries are best for journal discovery.
2. Transcript chunks are best for exact passage retrieval.

## Product Shape

1. Journal once, then get:
   - title
   - mood
   - topics
   - memorable quotes
   - key learnings
   - active questions
2. Revisit through multiple views:
   - date
   - topic
   - emotional pattern
   - active questions / open loops
3. Chat with an assistant that knows you, and can show its sources.

## Open Questions

1. How far should auto-organization go before it feels too opinionated?
2. When should a journal insight be promoted into long-term memory?
3. How should cross-journal threads be represented: tags, entities, links, or explicit timelines?

## Where AI Matters Most

1. Deep search
2. Memory extraction and promotion
3. Context-aware chat
4. Proactive stuff
